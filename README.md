# 🛡️ Πλαίσιο Διοίκησης και Ελέγχου (C2 Framework)

Μια εκπαιδευτική υλοποίηση ενός συστήματος απομακρυσμένου ελέγχου (Command & Control), σχεδιασμένη για την κατανόηση της αρχιτεκτονικής των σύγχρονων απειλών και των εργαλείων Red Teaming.

---

## 1. Γενική Εικόνα (Πώς λειτουργεί) 💡

Φανταστείτε μια εταιρεία με τέσσερις βασικούς "παίκτες":

* **👤 Εσείς, το Αφεντικό (Operator Console):** Κάθεστε στο γραφείο σας και δίνετε εντολές.
* **🏢 Η Γραμματεία (Team Server):** Ο ενδιάμεσος. Δεν μιλάτε απευθείας με τους υπαλλήλους· δίνετε τις εντολές στη Γραμματεία και αυτή τις προωθεί.
* **📖 Το Βιβλίο Καταγραφής (Database):** Εδώ καταγράφονται τα πάντα: ποιοι υπάλληλοι είναι ενεργοί, ποιες εντολές δόθηκαν και τι αποτελέσματα έφεραν.
* **🏃 Οι Υπάλληλοι (Agents/Implants):** Βρίσκονται "έξω" (στους υπολογιστές-στόχους). Κάθε λίγο επικοινωνούν με τη Γραμματεία και ρωτάνε: *"Έχει αφήσει το αφεντικό δουλειά για εμένα;"*.

### Η Ροή της Επικοινωνίας


1.  Ο **Agent** "ξυπνάει" στον στόχο και κάνει register στον Server.
2.  Ο **Operator** στέλνει μια εντολή (π.χ. "list files") μέσω της κονσόλας.
3.  Ο **Server** αποθηκεύει την εντολή στη βάση δεδομένων.
4.  Ο **Agent** κάνει "beacon" (επικοινωνεί), λαμβάνει την εντολή, την εκτελεί και επιστρέφει το αποτέλεσμα.
5.  **Ασφάλεια:** Η επικοινωνία προστατεύεται με κρυπτογράφηση **E2EE** και **Mutual Authentication**.

## 2. Αναλυτική Επεξήγηση Κώδικα 🛠️

Το project αποτελείται από τέσσερα βασικά αρχεία Python:

### 🗄️ 1. `database.py` (Το Βιβλίο Καταγραφής)
Διαχειρίζεται τη βάση δεδομένων **SQLite**.
* **Σκοπός:** Αποθήκευση των agents (IP, hostname, last seen) και των tasks (εντολή, status, output).
* **Τενολογία:** Χρήση `sqlite3` για ελαφριά και γρήγορη αποθήκευση χωρίς ανάγκη για εξωτερικό server βάσης.

### 📡 2. `server.py` (Διακομιστής)
Ο κεντρικός κόμβος του συστήματος βασισμένος σε **Flask**.
* **Authentication:** Χρησιμοποιεί `AUTH_TOKEN` στα headers για να επιτρέπει την πρόσβαση μόνο σε εξουσιοδοτημένους agents.
* **Encryption:** Χρησιμοποιεί τη βιβλιοθήκη `cryptography.fernet` για να κρυπτογραφεί τις εντολές πριν τις στείλει, ώστε να μην είναι αναγνώσιμες από τρίτους (IDS/IPS).

### 🕵️ 3. `agent.py` (Πελάτης)
Το script που εκτελείται στον υπολογιστή-στόχο.
* **Beaconing:** Επικοινωνεί ανά τακτά χρονικά διαστήματα με τον Server.
* **Jitter:** Εισάγει τυχαία καθυστέρηση (π.χ. 10s ± 2s) μεταξύ των επικοινωνιών για να αποφύγει την ανίχνευση από συστήματα που ψάχνουν για σταθερά "ρομποτικά" μοτίβα.
* **Execution:** Χρησιμοποιεί το `subprocess` για να τρέχει εντολές συστήματος και να επιστρέφει το output.

### 🎮 4. `console.py` (Κονσόλα Διαχείρησης)
Η διεπαφή του χρήστη (Operator).
* **Interactive Shell:** Μια CLI κονσόλα βασισμένη στη βιβλιοθήκη `cmd`.
* **Commands:** Επιτρέπει την προβολή των ενεργών agents (`agents`) και την ανάθεση εργασιών (`shell <command>`).

---
> **Προειδοποίηση:** Το παρόν λογισμικό προορίζεται αποκλειστικά για εκπαιδευτικούς σκοπούς και νόμιμες δοκιμές διείσδυσης με την άδεια των ιδιοκτητών των συστημάτων.
